# Timezone Bug Fix Summary

## Problem
The scheduler app was sending meeting reminders at incorrect times, off by exactly one hour, but only for meetings falling between 2am-3am local time (Europe/Be
rlin). This affected the Monday 2:30am maintenance window reminders.

## Root Cause
In `app.py`, the `convert_to_berlin()` function had a critical bug:

```python
# BUGGY CODE (line 28):
utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
berlin_dt = BERLIN_TZ.localize(utc_dt)  # WRONG!
```

The bug: `BERLIN_TZ.localize()` was being used on a UTC datetime, which incorrectly treated the UTC time as if it were already in Berlin timezone. This caused i
ssues during DST transitions (which occur at 2am-3am in Europe).

## Solution
Fixed the timezone conversion to properly convert FROM UTC TO Berlin:

```python
# FIXED CODE:
utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
utc_dt = UTC_TZ.localize(utc_dt)        # Mark as UTC
berlin_dt = utc_dt.astimezone(BERLIN_TZ)  # Convert to Berlin
```
