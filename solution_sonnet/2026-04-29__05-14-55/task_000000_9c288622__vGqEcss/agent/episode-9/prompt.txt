# Scheduler App DST Bug Fix

## Problem
The scheduler app was sending meeting reminders at incorrect times for meetings between 2am-3am local time (Europe/Berlin), with exactly 1 hour offset. This occ
urred specifically during DST transitions.

## Root Cause
In `app.py`, the `convert_to_berlin()` function had a critical bug:

```python
# BUGGY CODE (line 27):
utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
berlin_dt = BERLIN_TZ.localize(utc_dt)  # WRONG!
```

The bug: `BERLIN_TZ.localize()` was treating the naive UTC datetime as if it were already in Berlin time, instead of converting FROM UTC TO Berlin.

## Solution
Fixed the conversion to properly handle timezone-aware datetime objects:

```python
# FIXED CODE:
utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
utc_dt = UTC_TZ.localize(utc_dt)        # Make it UTC-aware first
berlin_dt = utc_dt.astimezone(BERLIN_TZ)  # Then convert to Berlin
```

## Impact
- The Monday 2:30am maintenance window meeting (stored as 2024-03-31T01:30:00 UTC) now correctly converts to 03:30 CEST instead of 01:30 CET
- During DST transition on March 31, 2024, when clocks spring forward from 2am to 3am, the conversion now properly accounts for this
- All timezone conversions now work correctly regardless of DST status
