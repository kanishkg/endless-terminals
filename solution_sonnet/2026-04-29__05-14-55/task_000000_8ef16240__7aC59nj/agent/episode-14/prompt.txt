> # Pipeline Checkpoint Resume Fix - Summary
>
> ## Bugs Found
>
> ### 1. **Critical: Checkpoint saved batch_num but treated as line number on resume**
>    - **Problem**: The script saved `batch_num` (number of batches processed) as 'last_line' in checkpoint
>    - **Impact**: If 5 batches (5,000 records) were processed, only 5 lines were skipped on resume!
>    - **Example**: Process 10,000 records (10 batches) → checkpoint saves 10 → resume skips only 10 records → reprocesses 9,990 records
>
> ### 2. **Critical: INSERT OR REPLACE overwrote data instead of accumulating**
>    - **Problem**: Used `INSERT OR REPLACE` which completely replaces existing rows
>    - **Impact**: On resume, previously aggregated totals were overwritten instead of accumulated
>    - **Result**: Final totals were incorrect, only reflecting the last run's data
>
> ### 3. **Minor: Checkpoint file not properly flushed**
>    - **Problem**: Checkpoint file opened without context manager, no explicit flush/fsync
>    - **Impact**: Could lose checkpoint data if process killed before OS flushes buffer
>
> ## Fixes Applied
>
> ### Fix 1: Track and save actual line number (records_processed)
> ```python
> # BEFORE:
> def save_checkpoint(batch_num):
>     json.dump({'last_line': batch_num}, open('.checkpoint', 'w'))
>
> # AFTER:
> def save_checkpoint(records_processed):
>     with open('.checkpoint', 'w') as f:
>         json.dump({'records_processed': records_processed}, f)
>         f.flush()
>         os.fsync(f.fileno())
> ```
>
> ### Fix 2: Use INSERT ... ON CONFLICT DO UPDATE to accumulate values
> ```python
> # BEFORE:
> conn.execute('INSERT OR REPLACE INTO summary VALUES (?, ?, ?)',
>              (cat, data['total'], data['count']))
>
> # AFTER:
> conn.execute('''
>     INSERT INTO summary (category, total_value, count)
>     VALUES (?, ?, ?)
>     ON CONFLICT(category) DO UPDATE SET
>         total_value = total_value + excluded.total_value,
>         count = count + excluded.count
> ''', (cat, data['total'], data['count']))
> ```
>
> ### Fix 3: Track records_processed correctly in the loop
> ```python
> # BEFORE:
> batch_num = 0

>         continue
>     # ... process ...
>     if len(batch) >= BATCH_SIZE:
>         process_batch(conn, batch)
>         batch_num += 1
>         save_checkpoint(batch_num)  # Wrong: saves batch count!
>
> # AFTER:
