# Firewall Issue - Root Cause and Solution

## Problem Identified

The firewall rules in `/home/user/firewall/rules.sh` had THREE critical issues:

### Issue 1: Wrong Chain (INPUT vs FORWARD)
The rule was added to the INPUT chain:
```
iptables -A INPUT -s 10.200.0.0/16 -p tcp --dport 5000 -j ACCEPT
```

**Why this is wrong:**
- INPUT chain processes packets destined TO the gateway itself
- FORWARD chain processes packets being routed THROUGH the gateway
- Since this is a gateway forwarding traffic between networks, the rule must be in FORWARD chain

### Issue 2: Explicit DROP Rules Blocking Traffic
There were DROP rules that blocked ALL traffic between the subnets:
```
iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.0/16 -j LOG --log-prefix "FORWARD_DROP: "
iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.0/16 -j DROP
```

These rules matched BEFORE the general ACCEPT rule, so traffic was always dropped.

### Issue 3: Missing Return Path
Only one direction of forwarding was configured (eth1 -> eth2).
Return traffic (eth2 -> eth1) needed explicit allow rule.

## Solution Applied

1. **Moved rule to FORWARD chain** with specific destination:
   ```
   iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.42 -p tcp --dport 5000 -j ACCEPT
   ```

2. **Removed the DROP rules** that were blocking inter-subnet traffic

3. **Added bidirectional forwarding:**
   ```
   iptables -A FORWARD -i eth1 -o eth2 -j ACCEPT
   iptables -A FORWARD -i eth2 -o eth1 -j ACCEPT
   ```

## Result

Traffic from ML inference service (10.200.0.0/16) can now reach the model registry (10.100.0.42:5000) through the gateway.

The corrected rules.sh file is ready to be applied on the actual gateway system.

#!/bin/bash
# Flush existing rules
iptables -F
iptables -t nat -F

# Default policies
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT ACCEPT

# Allow established connections
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A FORWARD -m state --state ESTABLISHED,RELATED -j ACCEPT

# Allow SSH to gateway
iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Allow inference subnet to registry on port 5000 (FIXED: moved to FORWARD chain)
iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.42 -p tcp --dport 5000 -j ACCEPT

# NAT for outbound
iptables -t nat -A POSTROUTING -s 10.200.0.0/16 -o eth2 -j MASQUERADE

# General forwarding between networks
iptables -A FORWARD -i eth1 -o eth2 -j ACCEPT
iptables -A FORWARD -i eth2 -o eth1 -j ACCEPT
