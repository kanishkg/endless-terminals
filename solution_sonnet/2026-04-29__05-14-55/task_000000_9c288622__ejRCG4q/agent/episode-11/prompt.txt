# Timezone Conversion Bug Fix

## Problem
The scheduler app was sending meeting reminders at incorrect times, with a 1-hour offset that was particularly noticeable for meetings between 2am-3am local tim
e (Europe/Berlin). This occurred during DST transitions.

## Root Cause
In `app.py`, the `convert_to_berlin()` function had a critical bug on line 28:

```python
# BUGGY CODE:
berlin_dt = BERLIN_TZ.localize(utc_dt)
```

This code was treating the parsed datetime (which represents a UTC time) as if it were already in Berlin timezone. The `localize()` method attaches a timezone t
o a naive datetime but doesn't convert it - it assumes the datetime is already in that timezone.

## The Fix
Changed lines 28-29 to properly convert FROM UTC TO Berlin:

```python
# FIXED CODE:
utc_dt = UTC_TZ.localize(utc_dt)      # Mark the datetime as UTC
berlin_dt = utc_dt.astimezone(BERLIN_TZ)  # Convert to Berlin time
```

## Why It Affected 2am-3am Meetings
The bug affected ALL meetings, but was most noticeable during DST transitions:
- March 31, 2024: DST starts in Europe/Berlin (clocks spring forward from 2am to 3am)
- The "Maintenance Window Prep" meeting at 01:30 UTC should be 03:30 CEST
- The buggy code converted it to 01:30 CET (wrong timezone, wrong time)
- This made the 2am-3am window particularly problematic

## Verification
Run `python3 verify_fix.py` to see the before/after comparison for all meetings.
