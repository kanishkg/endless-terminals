=== INCREMENTAL BACKUP CORRUPTION FIX ===

PROBLEM:
Incremental backups were producing corrupt archives with "unexpected EOF" errors
during extraction, while full backups worked fine.

ROOT CAUSE:
The incremental.sh script had a race condition:
- Line 13: tar was run in background with '&'
- Line 15: timestamp was updated IMMEDIATELY (before tar finished)
- Line 16: script waited for tar to complete

This caused the timestamp to be updated while tar was still reading from stdin,
potentially causing stdin to close prematurely or data to be lost, resulting in
corrupt/truncated archives.

SOLUTION:
Removed the backgrounding of tar process:
- Removed '&' from tar command (line 13)
- Removed TAR_PID variable capture (line 14)
- Removed 'wait $TAR_PID' (line 16)
- Kept timestamp update after tar completes (line 15)

Now tar runs synchronously, and timestamp is only updated after successful completion.
The 'set -e' ensures if tar fails, the timestamp won't be updated.

- Created test incremental backups
- Successfully extracted without errors (exit code 0)
- All files present and uncorrupted
- Subsequent incremental backups also work correctly
