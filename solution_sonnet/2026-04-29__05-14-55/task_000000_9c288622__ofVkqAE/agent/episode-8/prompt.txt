=============================================================================
BUG FIX SUMMARY: Meeting Reminder Timezone Conversion Issue
=============================================================================

PROBLEM:
--------
Meeting reminders were being sent at incorrect times (off by exactly 1 hour)
for meetings scheduled between 2am-3am Berlin time. This occurred when the
app was run via cron but worked correctly during manual testing.

ROOT CAUSE:
-----------
The convert_to_berlin() function in app.py was incorrectly using:
    BERLIN_TZ.localize(utc_dt)

This treated the naive datetime (parsed from UTC string) as if it were
already in Berlin timezone, rather than converting FROM UTC TO Berlin.

During DST transitions (like March 31, 2024 when clocks spring forward),
this caused a 1-hour offset error for meetings in the 2am-3am window.

SOLUTION:
---------
Fixed the convert_to_berlin() function to properly:
1. Localize the naive datetime as UTC: UTC_TZ.localize(utc_dt)
2. Convert to Berlin timezone: utc_dt.astimezone(BERLIN_TZ)
