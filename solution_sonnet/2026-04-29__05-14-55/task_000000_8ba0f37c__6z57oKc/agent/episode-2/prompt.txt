=== HTTP Traffic Blocking Analysis ===

BLOCKING RULES IDENTIFIED:
Rules 7-10 in the iptables configuration are blocking HTTP/HTTPS traffic from 10.0.0.5:

Rule 7: -A INPUT -p tcp --dport 80 -m string --string "Mozilla" --algo bm -j ACCEPT
Rule 8: -A INPUT -p tcp --dport 443 -m string --string "Mozilla" --algo bm -j ACCEPT
Rule 9: -A INPUT -p tcp --dport 80 -j DROP
Rule 10: -A INPUT -p tcp --dport 443 -j DROP

WHY HTTP IS BLOCKED BUT SSH IS NOT:

1. SSH (port 22) has an unconditional ACCEPT rule:
   -A INPUT -p tcp --dport 22 -j ACCEPT
   This allows ALL SSH traffic regardless of packet content.

2. HTTP/HTTPS (ports 80/443) use conditional string matching:
   - Traffic is only accepted if the packet payload contains "Mozilla"
   - The string match module inspects packet data using Boyer-Moore algorithm
   - If "Mozilla" is NOT found, traffic falls through to the DROP rules

3. Why curl hangs:
   - curl's default User-Agent is "curl/X.X.X", not "Mozilla/..."
   - The string "Mozilla" is not present in curl's HTTP requests
   - Traffic doesn't match the ACCEPT rule, hits the DROP rule instead
   - TCP connection hangs because packets are silently dropped after the handshake

TECHNICAL ISSUE:
The string matching approach is fundamentally flawed for access control:
- It attempts to filter by User-Agent string in application-layer data
- This breaks legitimate tools (curl, wget, scripts, APIs, monitoring tools)
- String matching on packet payloads is unreliable and easily bypassed
- It creates a false sense of security while blocking legitimate traffic

RECOMMENDATION:
Remove the string-based filtering rules (7-10) and replace with proper access control:
- Use source IP-based filtering if restricting access by origin
- Implement rate limiting with the 'recent' or 'limit' modules at the netfilter level
- Move application-layer filtering to the web server (nginx/apache) where it belongs
- If User-Agent filtering is truly needed, handle it in the application, not iptables

The current ruleset blocks all HTTP clients except those with "Mozilla" in their User-Agent,
which is not a security control but rather an overly restrictive filter that breaks
legitimate access while being trivially bypassable.
