FIREWALL FIX SUMMARY
====================

PROBLEM:
Traffic from ML inference service (10.200.0.0/16) to model registry (10.100.0.42:5000)
was timing out despite having an ACCEPT rule visible in iptables -L.

ROOT CAUSES:
1. The ACCEPT rule was in the INPUT chain instead of FORWARD chain
   - INPUT chain only applies to traffic destined FOR the gateway itself
   - FORWARD chain applies to traffic being ROUTED THROUGH the gateway
   - Since this is a gateway routing between eth1 and eth2, FORWARD is required

2. Explicit DROP rules were blocking ALL traffic from 10.200.0.0/16 to 10.100.0.0/16
   - These rules came AFTER the comment "Block known bad actors" (misplaced)
   - They were positioned BEFORE the general forwarding rules
   - Rule ordering matters: first match wins

FIXES APPLIED TO /home/user/firewall/rules.sh:
1. Moved rule from INPUT to FORWARD chain:
   OLD: iptables -A INPUT -s 10.200.0.0/16 -p tcp --dport 5000 -j ACCEPT
   NEW: iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.42 -p tcp --dport 5000 -j ACCEPT

2. Removed blocking DROP rules:
   REMOVED: iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.0/16 -j LOG --log-prefix "FORWARD_DROP: "
   REMOVED: iptables -A FORWARD -s 10.200.0.0/16 -d 10.100.0.0/16 -j DROP

3. Added bidirectional forwarding:
   ADDED: iptables -A FORWARD -i eth2 -o eth1 -j ACCEPT (for return traffic)

RESULT:
