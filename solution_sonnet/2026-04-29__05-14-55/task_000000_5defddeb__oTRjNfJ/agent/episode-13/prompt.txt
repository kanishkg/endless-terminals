PROBLEM:
--------
Incremental backups were producing corrupt archives with "unexpected EOF" errors
and truncated files. Full backups worked fine.

ROOT CAUSE:
-----------
The incremental.sh script was backgrounding the tar process (&) and updating
the state files (timestamp and reference) BEFORE tar completed. This caused:

1. Race condition: State updated while tar still writing
2. If tar failed/interrupted after state update, files lost from future incrementals
3. Corrupted archives due to premature state updates

THE BUG (lines 13-16 of original script):
    tar --null -T - -czf "$ARCHIVE_DIR/$ARCHIVE_NAME" &
    TAR_PID=$!
    date +%s > "$TIMESTAMP_FILE"  # <-- Updated BEFORE tar completes!
    wait $TAR_PID

THE FIX:
--------
Removed backgrounding and moved state updates to AFTER tar completes:
    tar --null -T - -czf "$ARCHIVE_DIR/$ARCHIVE_NAME"
    date +%s > "$TIMESTAMP_FILE"  # <-- Updated AFTER tar completes
