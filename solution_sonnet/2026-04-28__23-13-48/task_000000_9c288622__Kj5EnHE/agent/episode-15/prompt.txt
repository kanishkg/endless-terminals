> # Timezone Bug Fix Summary
>
> ## Problem
> The meeting reminder application was sending reminders at incorrect times (off by 1 hour) for meetings scheduled between 2am-3am local time (Europe/Berlin) wh
en run via cron, but worked correctly when tested manually.
>
> ## Root Cause
> The bug was in the `convert_to_berlin()` function in `app.py`:
>
> **Buggy code (line 28):**
> ```python
> utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
> berlin_dt = BERLIN_TZ.localize(utc_dt)  # WRONG!
> ```
>
> This code:
> 1. Parses a UTC time string into a naive (timezone-unaware) datetime
> 2. Uses `BERLIN_TZ.localize()` which treats the naive datetime as if it's already in Berlin time
> 3. This is incorrect because the datetime is actually in UTC, not Berlin time
>
> ## The Fix
> **Corrected code:**
> ```python
> utc_dt = datetime.strptime(utc_time_str, '%Y-%m-%dT%H:%M:%S')
> utc_dt_aware = UTC_TZ.localize(utc_dt)  # First, mark it as UTC
> berlin_dt = utc_dt_aware.astimezone(BERLIN_TZ)  # Then convert to Berlin
> ```
>
> This correctly:
> 1. Parses the UTC time string
> 2. Localizes it as UTC using `UTC_TZ.localize()`
> 3. Converts from UTC to Berlin timezone using `astimezone()`
>
> ## Why It Only Failed During DST Transitions
> The bug was most apparent during DST transitions (2am-3am window):
> - On March 31, 2024, Europe/Berlin transitions from CET to CEST at 2am
> - Clocks spring forward from 2:00am to 3:00am
> - The meeting at UTC 01:30:00 should convert to CEST 03:30:00
> - The buggy code converted it to CET 01:30:00 (wrong by 2 hours in absolute terms, 1 hour in local time)
>
