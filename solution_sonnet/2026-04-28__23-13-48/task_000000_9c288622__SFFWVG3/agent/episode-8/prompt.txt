BUG FIX SUMMARY
===============

Problem:
--------
The scheduler was sending meeting reminders at incorrect times (off by 1 hour)
for meetings between 2am-3am local time, specifically during DST transitions.

Root Cause:
-----------
In app.py, the convert_to_berlin() function was using:
  berlin_dt = BERLIN_TZ.localize(utc_dt)

This incorrectly treated the naive datetime (which represented UTC) as if it
were already in Berlin time, then just added timezone info.

The Fix:
--------
Changed to the correct two-step conversion:
  utc_dt_aware = UTC_TZ.localize(utc_dt)      # Mark as UTC
  berlin_dt = utc_dt_aware.astimezone(BERLIN_TZ)  # Convert to Berlin

This properly handles DST transitions.

Example (Maintenance Window Prep meeting):
------------------------------------------
UTC Time: 2024-03-31 01:30:00

OLD (buggy):  2024-03-31 01:30:00+01:00 CET  (Wrong!)
NEW (fixed):  2024-03-31 03:30:00+02:00 CEST (Correct!)

On March 31, 2024, Berlin springs forward to DST at 2am, so UTC+01:30
correctly becomes Berlin 03:30 CEST (UTC+2).

Files Modified:
--------------
- /home/user/scheduler/app.py (fixed convert_to_berlin function)
- Backup saved as app.py.backup

The crontab remains unchanged and will now work correctly.
